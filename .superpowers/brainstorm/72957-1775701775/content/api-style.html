TextInput API 风格选择

如何将响应式状态与解耦合的 TextInputClient 绑定

A

信号驱动 (Xilem/Leptos 风格)

极致简洁。 开发者只需提供一个 Signal。所有复杂的 FFI 同步和事务处理都封装在组件内部。

let name = create_signal("World".to_string());

TextInput {
  text: name,
  on_change: move |val| name.set(val),
}
  • 声明式代码，易于上手
  • 符合 Dyxel 目前的信号系统
  • 很难精细控制光标和选区
  • 难以实现复杂的自定义逻辑 (如高亮)
B

控制器驱动 (Flutter 风格)

精细控制。 开发者创建一个 Controller。Controller 包含 Text、Selection 和 Composing 状态，并管理所有事务。

let controller = use_text_controller("World");

TextInput {
  controller: controller,
}

// 可以在外部精细操作
controller.select_all();
controller.insert_at_cursor("!");
  • 强大的功能 (光标位置、多行选择)
  • 支持富文本、语法高亮等高级功能
  • API 相对繁琐
  • 需要管理 Controller 的生命周期
底层机制：FFI 事务流
  1. Native IME 捕获按键
  2. 发送 TextUpdateTransaction 到 WASM
  3. WASM Controller 调和状态 (Reconciliation)
  4. 触发 Signal 更新或重绘视图
