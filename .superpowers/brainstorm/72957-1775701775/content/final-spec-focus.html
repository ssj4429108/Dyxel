TextInput 深度设计：响应式、事务与焦点系统

融合 Xilem 响应式理念与 Flutter 解耦合架构的完整设计

1. 混合事务架构 (The Hybrid Engine)

采用解耦合模型。Native (Host) 仅作为 IME 代理，WASM (Guest) 作为状态真相。二者通过 TextTransaction 异步同步。

  • Native: 处理按键、联想词、选区手势，发送 Delta。
  • WASM: 校验 Delta，更新 Signal，计算 Vello 渲染参数。

2. 响应式值 API (Reactive Value)

放弃简单的 String 信号，引入打包的 TextState。支持光标、选区、组合状态的完整响应式追踪。

let state = create_signal(TextState::new(""));
TextInput {
  value: state,
  on_change: |val| state.set(val),
}

3. 焦点抢占系统 (Focus Preemption)

核心改进： 引入全局 FocusManager，支持声明式焦点切换与抢占。

  • 自动抢占： 点击新节点自动触发旧节点 Blur 事务同步。
  • FocusScope： 支持局部焦点隔离（如弹窗内）。
  • Native 同步： 焦点切换即时同步键盘类型（数字/文本）。

4. WASM 像素级渲染 (Vello Render)

Native 仅负责输入，不负责显示。Vello 渲染所有视觉元素，保证全平台一致性。

  • 自定义闪烁光标（支持平滑呼吸动画）。
  • 高性能选区高亮（半透明遮罩）。
  • IME 组合区下划线渲染。
